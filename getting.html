Getting Started

So, now that you’re here, and you know what we believe (check out Philosophy page if you don’t know)...

How can you get started helping Everest quickly?

First, let’s have a chat about where we use Open Source tools.

At Everest, we believe in developing “Openly Proprietary” solutions which leverage open source software in a proprietary software stack. Why do we do this - it is truly the best of both closed source and open source software. We are strong proponents of the open source software movement, and contribute back to those projects we utilize, however, there are certain elements of our solution which are so sensitive that we should keep those secret.

This is a very similar approach to Apple Inc. They built their operating system above BSD, a well known open source project, however, included a series of innovations above BSD which they have kept private. They continue to contribute to open source projects that they utilize, but do not release all of their software under Open Source licenses.

So, speaking of the complete software stack shared on the Architecture page, we will have mobile application software, server software, web software, testing software and security software.

Our smart contracts are written in a standard language, Solidity, however, we are running a private, permissioned blockchain network so no one else has access to these smart contracts. While we could open source them, there is no benefit to the community (they are very specific to our infrastructure), nor is there benefit to Everest (they are not open to the abuse of an open-network smart contract).

Types of OSS we do:

Open Source Projects - outside projects that we use

Our Tools - tools which we create

Our Testing - testing materials we create

Our Modules - reusable software modules we make
